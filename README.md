# Tasteorama

> Рецепти, які не губляться. Пошук, фільтри, збережене — і кукі-сесія, що переживає деплой.

Фронтенд застосунку рецептів на **Next.js 15** (App Router): серверні компоненти за
замовчуванням, кукі-автентифікація без жодного токена в JS, адаптив від 320 до 1440.
Бекенд живе окремо (Express + MongoDB на Render) — цей репозиторій лише розмовляє з ним
по HTTP.

**Демо:** https://six-seven-company-frontend.vercel.app

---

## Що всередині

- **Пошук і фільтри** — за назвою, категорією та інгредієнтом. Зміна будь-якого фільтра
  завжди повертає на першу сторінку — так, як і має бути.
- **Кукі-сесія** — `accessToken` / `refreshToken` / `sessionId`, нічого не валяється в
  localStorage. Сесія підхоплюється на старті й тихо оновлюється сама.
- **Збережене та власні рецепти** — профіль на дві вкладки з лічильником і миттєвим
  оновленням списку після кожної дії, без перезавантаження.
- **Додавання рецепту** — форма із завантаженням фото (multipart), валідація на
  Formik + Yup, інлайнові помилки під полями.
- **Адаптив 320 → 768 → 1440** — три брейкпоінти, mobile-first, без проміжних милиць.
- **Зворотний зв'язок** — лоадер на кожен активний запит, тости на успіх і помилку,
  модалки там, де неавторизований користувач тицяє «зберегти».

## Одне правило про оточення — прочитай, зекономить вечір

Застосунок ходить у бекенд **через власний проксі** Next.js (`app/api/[...path]/route.ts`).
Це не примха: так кукі ставиться на домен фронтенду, і серверний `middleware.ts` її бачить.

- `NEXT_PUBLIC_API_URL` → **лишай порожнім.** Тоді axios б'є у відносний `/api`, і всі
  запити йдуть крізь проксі.
- `API_URL` → базова адреса бекенду (напр. `https://six-seven-company.onrender.com`). Її
  використовує проксі для звернень сервер-до-сервера.

> Виставиш `NEXT_PUBLIC_API_URL` на бекенд напряму — кукі осяде на чужому домені, клієнт
> ніби залогінений, а `/profile` усе одно кидає на логін. На деплої цю змінну **прибери**
> і зроби redeploy: `NEXT_PUBLIC_*` вшивається у бандл ще на етапі білду.

## Як це складено

```
app/          маршрути App Router + катч-ол проксі (api/[...path]/route.ts)
components/    компонент = тека: Component.tsx + Component.module.css
lib/api/       clientApi (браузер) · serverApi (сервер) · edgeAuth (middleware, на fetch)
lib/store/     Zustand: authStore, filtersStore
types/         спільний контракт даних
```

Потік запиту коротко: `компонент → clientApi → axios (/api) → проксі → бекенд`.
